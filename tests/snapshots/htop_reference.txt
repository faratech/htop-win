## CPU Bar Format:
   N[|||||...     XX.X%]
   │ │           │
   │ │           └── Percentage (right-aligned, 5 chars + %)
   │ └── Bar fill (| chars) + empty spaces
   └── CPU number (right-aligned, 3 chars)

## Colors:
   - CPU bar < 50%: GREEN
   - CPU bar 50-80%: YELLOW
   - CPU bar > 80%: RED
   - Memory bar: GREEN to BLUE gradient
   - Swap bar: Same as memory
   - Process Running (R): GREEN
   - Process Sleeping (S): Normal/GRAY
   - Process Zombie (Z): RED
   - Process Stopped (T): YELLOW
   - Selected row: INVERSE (black on cyan)
   - Tagged process: YELLOW text

## Column Widths (approximate):
   PID:     7 chars (right-aligned)
   USER:   10 chars (left-aligned)
   PRI:     4 chars (right-aligned)
   NI:      4 chars (right-aligned)
   VIRT:    7 chars (right-aligned, human-readable)
   RES:     7 chars (right-aligned, human-readable)
   SHR:     7 chars (right-aligned, human-readable)
   S:       1 char  (centered)
   CPU%:    5 chars (right-aligned, 1 decimal)
   MEM%:    5 chars (right-aligned, 1 decimal)
   TIME+:  10 chars (right-aligned, MM:SS.cc or HH:MM:SS)
   Command: remaining width (left-aligned)

## Tree View Prefixes:
   ├─ child process (not last)
   └─ child process (last)
     ├─ grandchild (indent 2 spaces per level)

## Function Key Bar:
   F1Help  F2Setup F3Search F4Filter F5Tree  F6SortBy F7Nice- F8Nice+ F9Kill  F10Quit
   │       │
   │       └── Label (normal text)
   └── Key (inverse/highlighted)

## Header Section:
   - CPUs displayed in 2 columns
   - Memory bar spans full width
   - Swap bar spans full width
   - Tasks line: "Tasks: N, M thr; K running"
   - Uptime line: "Uptime: DD:HH:MM:SS" or "HH:MM:SS"

## Footer (2 lines):
   - Line 1: Function keys F1-F10
   - Line 2: Status info (filter, search, sort indicator)
